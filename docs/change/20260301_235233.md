## Details

### Added: `TestYfinanceFetchUSStocks` test class (4 new test cases)

Since `STOCK_LIST` contains US stocks (`NVDA`, `TSLA`) that Tushare cannot fetch, added a dedicated test class using `YfinanceFetcher` to verify US stock data availability.

| Test | Purpose |
|---|---|
| `test_yfinance_installed` | Verify `yfinance` package is available in the environment |
| `test_yfinance_ticker_history` | Call `yfinance.Ticker().history()` directly for each US stock to validate network connectivity and data |
| `test_yfinance_fetcher_integration` | End-to-end test via project's `YfinanceFetcher.get_daily_data()` |
| `test_yfinance_fetcher_realtime_quote` | Test `YfinanceFetcher.get_realtime_quote()` for US stock realtime data |

### Updated: Standalone runner

- Added step `[4/5]`: raw yfinance data download for US stocks
- Added step `[5/5]`: `YfinanceFetcher` integration test for US stocks
- Updated summary section to show data source routing (A-share via Tushare, US via yfinance)

### Design notes

- `YfinanceFetcher` is imported via `importlib.util` (same isolation pattern as `TushareFetcher`) to avoid triggering `data_provider/__init__.py` and its transitive dependencies
- Used `Ticker.history()` instead of `yf.download()` for the raw API test â€” more reliable in pytest context where `yf.download()` can hit internal state issues
- `us_stock_list` pytest fixture filters `STOCK_LIST` to US-only codes automatically
