## Details

### New: `tests/test_tushare_stock_fetch.py`

Added a comprehensive test script to verify whether the `TUSHARE_TOKEN` configured in `.env` can successfully fetch stock data for stocks in `STOCK_LIST`.

**Test structure (3 test classes, 7 test cases):**

| Class | Test | Purpose |
|---|---|---|
| `TestTushareTokenValidity` | `test_token_is_configured` | Verify TUSHARE_TOKEN is present and has reasonable length |
| | `test_api_initializes` | Verify `tushare.pro_api()` initializes without error |
| | `test_api_reachable` | Call `daily(600519.SH)` to confirm token is accepted by server |
| `TestStockListFetch` | `test_stock_list_not_empty` | Verify STOCK_LIST is not empty |
| | `test_fetch_each_stock` | Iterate STOCK_LIST, fetch via raw Tushare API; gracefully skip US stocks |
| `TestTushareFetcherIntegration` | `test_fetcher_initialization` | Verify project's `TushareFetcher` class initializes correctly |
| | `test_fetcher_get_daily_data` | End-to-end test using `TushareFetcher.get_daily_data()` |

**Key design decisions:**

- Uses `requests.post` to call Tushare API directly (avoids SDK endpoint issues)
- Uses `daily` API instead of `trade_cal` for connectivity check (lower permission requirement)
- Imports `TushareFetcher` via `importlib.util` to bypass `data_provider/__init__.py` which pulls in all fetchers and their transitive dependencies
- US stock codes (e.g. NVDA, TSLA) are detected and gracefully skipped with clear log messages
- Supports both `pytest` and standalone execution modes

## Test Result

- TUSHARE_TOKEN: Valid, API reachable
- STOCK_LIST (`NVDA, TSLA`): All US stocks — correctly skipped by Tushare, need YfinanceFetcher
